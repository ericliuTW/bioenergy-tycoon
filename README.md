# 🌱 生質能校園大亨：微電網保衛戰

**Bioenergy Tycoon: Microgrid Defense**

一款針對高中生設計的 2D 網頁互動式經營模擬遊戲，讓學生透過遊戲理解生質能系統的核心概念。

---

## 🎮 如何開始

### 方法一：直接開啟（推薦）
1. 在 Chrome 瀏覽器中直接開啟 `index.html`
2. 不需要安裝任何軟體或啟動伺服器

### 方法二：本地伺服器
如果遇到字型載入問題，可使用簡易伺服器：
```bash
# Python 3
python -m http.server 8000

# Node.js
npx serve .
```
然後打開 `http://localhost:8000`

## 📖 遊戲簡介

### 玩家身分
校園永續能源規劃者，負責為花蓮某高中規劃生質能系統。

### 遊戲目標
- 在 5 分鐘（300 秒）內建立完整的生質能處理與發電系統
- 將校園廢棄物（廚餘、枯枝、糞水）轉化為電力
- 控制外部成本（臭味、噪音、汙染），避免社區抗議
- 在颱風停電事件中維持校園微電網供電

### 操作方式
| 操作 | 說明 |
|------|------|
| 點擊綠色空地 | 開啟建設選單，放置設備 |
| 拖曳（從資源/設備到另一設備） | 建立物流管線 |
| 點擊資源或設備 | 查看詳細資訊 |
| 暫停按鈕 | 暫停/繼續遊戲 |

## 🔬 核心教學概念

### 1. 生質資源辨識
- **學生廚餘**：C/N = 12，高含水，適合厭氧消化但單獨使用易氨抑制
- **枯枝落葉**：C/N = 80，乾燥木質，作為碳源調整 C/N 比
- **農場糞水**：C/N = 8，高含水高臭味，需妥善處理
- **廢食用油**：C/N = 50，高產氣潛力

### 2. C/N 比與共消化
- 最佳 C/N 比：**20-30**
- C/N < 15：氨抑制，效率大幅下降
- C/N > 35：氮不足，微生物生長緩慢
- **關鍵學習**：混合多種資源可優化 C/N 配比

### 3. 設備處理鏈
```
資源 → 前處理設備 → 均質混合槽 → 厭氧消化槽 → 生物脫硫塔 → 沼氣發電機
                                                    ↑
                                              廢氣洗滌塔（降低臭味）
```

### 4. pH / OLR 穩定度
- pH 初始值 7.0，過度餵料會導致 pH 下降
- pH < 6.0 觸發酸化崩潰，產氣停止
- 可投入鹼劑恢復（需花費資金）

### 5. 外部成本
- 臭味、噪音、汙染累積至 100% 觸發抗議，遊戲失敗
- 脫硫塔、洗滌塔、短物流路線可有效控制

### 6. 微電網韌性
- 遊戲進行至剩餘 150 秒時觸發颱風停電
- 若發電系統穩定運作，可進入孤島供電模式獲得大量加分

## 🏫 教師上課使用建議

### 適合課程
- 高中自然科學探究與實作
- 能源教育
- 永續發展議題
- 地球科學（能源單元）

### 建議活動流程（一節課 45 分鐘）

| 時間 | 活動 |
|------|------|
| 0-5 分鐘 | 教師簡介生質能概念，說明遊戲目標 |
| 5-10 分鐘 | 學生閱讀新手教學，了解操作 |
| 10-15 分鐘 | 第一輪遊戲（探索期） |
| 15-20 分鐘 | 全班討論：為什麼你的系統崩潰/成功？ |
| 20-25 分鐘 | 教師補充 C/N 比、厭氧消化、外部成本概念 |
| 25-35 分鐘 | 第二輪遊戲（策略優化） |
| 35-45 分鐘 | 比較兩次分數，討論學習診斷報告 |

### 課堂討論引導問題
1. 為什麼只用廚餘的系統容易出問題？（C/N 比太低 → 氨抑制）
2. 加入枯枝落葉後 C/N 比如何變化？（碳源增加，C/N 上升）
3. 不建脫硫塔會怎樣？（發電機腐蝕、效率下降）
4. 物流路線長短如何影響成本？（距離越長，成本和臭味越高）
5. 停電時生質能發電為什麼重要？（能源韌性、分散式電力）

### 觀察重點
- 學生是否嘗試混合不同資源
- 學生是否注意到 C/N 比變化
- 學生是否理解「便宜配置」的長期代價
- 學生在第二輪遊戲中是否改進策略

## ⚙️ 參數調整指南

所有可調參數集中在 `js/config.js` 的 `GameConfig` 物件中。

### 常用調整

| 參數 | 位置 | 說明 | 預設值 |
|------|------|------|--------|
| `GAME_DURATION` | 頂部 | 遊戲時長（秒） | 300 |
| `INITIAL_MONEY` | 頂部 | 初始資金 | 55000 |
| `INITIAL_MANPOWER` | 頂部 | 初始人力 | 10 |
| `POWER_OUTAGE_TIME` | 頂部 | 停電觸發時間（剩餘秒） | 150 |
| 各設備 `cost` | `BUILDINGS` 區 | 設備價格 | 各異 |
| `CN_OPTIMAL_MIN/MAX` | C/N 區 | 最佳 C/N 範圍 | 20-30 |
| `PH_MIN_SAFE` | pH 區 | 酸化門檻 | 6.0 |
| `EXTERNALITY.MAX_PROTEST` | 外部成本區 | 抗議值上限 | 100 |
| `SCORING` 各權重 | 分數區 | 評分權重 | 見檔案 |

### 降低難度
- 增加 `INITIAL_MONEY`（如 80000）
- 增加 `GAME_DURATION`（如 420）
- 降低 `EXTERNALITY.SMELL_PER_TICK_BASE`
- 降低 `FAILURE.NO_PREPROCESS_CLOG_RATE`

### 提高難度
- 減少 `INITIAL_MONEY`（如 40000）
- 增加外部成本累積速度
- 提高故障機率
